 note about creating pods using kubectl run.

To create a pod from the command line, use the command:

Create an NGINX Pod

kubectl run nginx --image=nginx

As of version 1.18, kubectl run (without any arguments such as --generator ) will create a pod instead of a deployment.

To create a deployment using imperative command, use kubectl create:

kubectl create deployment nginx --image=nginx

/////////////////////////////////////Replication Controllers and Replica Set ////////

ReplicationController and ReplicaSet are both Kubernetes mechanisms that ensure a certain number of Pods are always running, but they are not equal. One is legacy, one is modern.

ReplicationController → old, deprecated, almost nobody uses it today.

ReplicaSet → new, more powerful, supports label selectors, used internally by Deployments.


The real-world rule
You never create ReplicaSets manually.
You always use a Deployment, which manages ReplicaSets for you.

Example:kubectl create deployment redis --image=redis

Why it matters
Deployments use ReplicaSets internally.

Supports rolling updates (through Deployment).

More flexible and modern.

Example ReplicaSet YAML

apiVersion: apps/v1
kind: ReplicaSet
metadata:
  name: redis-rs
spec:
  replicas: 3
  selector:
    matchLabels:
      app: redis
  template:
    metadata:
      labels:
        app: redis
    spec:
      containers:
        - name: redis
          image: redis


See all ReplicaSets in the cluster:
kubectl get rs


Key fields:

DESIRED → how many Pods you want

CURRENT → how many Pods exist

READY → how many Pods are healthy

If READY = DESIRED, your ReplicaSet is running correctly.


Describe a specific ReplicaSet

kubectl describe rs redis-rs


Check the Pods created by the ReplicaSet

kubectl get pods -l app=redis


Check ReplicaSet status via Deployment (if applicable)

kubectl get deployment
kubectl describe deployment my-deploy


Quick summary of what each does
Command	Purpose
kubectl create -f	Creates a ReplicaSet from the YAML file.
kubectl get replicaset	Lists all ReplicaSets in the current namespace.
kubectl delete replicaset	Deletes a specific ReplicaSet by name.
kubectl replace -f	Replaces an existing ReplicaSet with the updated YAML definition.
kubectl scale --replicas=6 -f	Changes the number of replicas to 6 using the YAML file.


If you want to delete the ReplicaSet but keep the Pods

kubectl delete rs myapp-rs --cascade=orphan
Result:
ReplicaSet is deleted

Pods remain running

Pods become “orphaned” (no controller manages them)

This is useful for debugging or migration.



Action	ReplicaSet	Pods	Nodes
kubectl delete rs myapp-rs	❌ deleted	❌ deleted	✔️ unaffected
kubectl delete rs myapp-rs --cascade=orphan	❌ deleted	✔️ kept	✔️ unaffected
